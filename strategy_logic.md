# 策略逻辑整理

## 一、数据层

### 日K线数据 (`binance_daily_klines`)
- **来源**: 币安现货 + 合约 API
- **周期**: 1日 (1d)
- **存储**: PostgreSQL
- **覆盖**: 430 现货USDT + 529 合约USDT (共959个交易对)
- **更新**: 每日00:05定时增量获取

---

## 二、筛选逻辑

### 超跌反弹筛选 (`oversold_reversal_scanner.py`)

**目的**: 找出跌得够深、够久，但短期开始转强的币种

| 条件 | 参数 | 说明 |
|------|------|------|
| 回撤深度 | `--drawdown 0.80` | 从历史最高价回撤 ≥ 80% |
| 弱势时长 | `--weak-days 120` | 过去N天中 ≥80% 天数收盘价低于 **60周均线**(≈300日均线) |
| 均线突破 | 收盘价 > 日线MA6 且 > 日线MA24 | 当前日线收盘价同时站上6日和24日均线 |

**输出**: 候选币种列表

```bash
# 示例：扫描现货，回撤85%以上，弱势120天
/data/anaconda3/envs/okx_api/bin/python3 oversold_reversal_scanner.py \
    --market spot --drawdown 0.85 --weak-days 120
```

## 三、完整数据流

```
币安API
    │
    └─→ 日K线 (1d) ──→ PostgreSQL (binance_daily_klines)
                           │
                           ├─→ 计算日线MA6 / MA24 / 60周均线
                           │
                           └─→ 超跌筛选
                                  │
                                  └─→ 候选币种列表
```

---

## 四、各程序文件

| 文件 | 功能 | 运行方式 |
|------|------|---------|
| `binance_daily_klines_pg.py` | 获取日K线，存入PostgreSQL | 每日定时 (crontab/systemd) |
| `oversold_reversal_scanner.py` | 日线级别超跌+均线突破筛选 | 手动运行，看候选币种 |

---

## 五、可调参数

### 超跌扫描器参数

```
--market    spot/swap     市场类型
--drawdown  0.0~1.0       回撤阈值 (默认0.80 = 跌80%)
--weak-days 整数          低于长期均线天数 (默认250≈1年)
--short-ma  整数          短期均线 (默认6日)
--medium-ma 整数          中期均线 (默认24日)
```

### 参数影响

| 参数调紧 | 效果 |
|---------|------|
| `--drawdown 0.90` | 只留跌90%以上的，更极端 |
| `--weak-days 250` | 要求弱势时间更长 |
| `--weak-days 120` | 放宽到半年，候选更多 |

---

## 六、注意事项

1. **数据 freshness**: 扫描器依赖日K线数据，需先确保 `binance_daily_klines_pg.py` 已运行到最新日期
2. **均线滞后性**: MA6/MA24 基于历史收盘价，有天然滞后
3. **信号频率**: 日线级别信号稀疏，通常一天只出一批结果
4. **回撤计算**: 使用回看窗口内（默认500天）的最高价，不是绝对历史最高价
